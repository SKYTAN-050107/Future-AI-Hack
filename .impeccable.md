# Design Context — PadiGuard AI

## Product
PadiGuard AI is a mobile-first, offline-capable PWA that helps Malaysian smallholder padi farmers detect crop disease, assess severity, and take actionable treatment steps using AI.

## Core Goal
Increase app installs and user trust by presenting clear, actionable, and reliable information.

---

## Target Users

### Primary
Smallholder padi farmers in Malaysia:
- Low to moderate digital literacy
- Use mobile devices (Android dominant)
- Often operate in low-connectivity environments

### Key Needs
- Extremely simple UI
- Clear “what to do next” guidance
- Fast understanding (low cognitive load)
- Trustworthy, non-technical language

---

## UX Priorities (CRITICAL)

1. Clarity over visual effects
2. One primary action per screen
3. Reduce cognitive load
4. Fast scanning (users should understand within 3 seconds)
5. Trust and credibility > “cool UI”

---

## Design Direction

### Tone
- Practical
- Calm
- Trustworthy
- Grounded (real farming context)
- Not “startup flashy” or overly techy

### Visual Style
- Clean mobile-app style UI
- Card-based layout
- Strong spacing and hierarchy
- Minimal decoration

---

## STRICT DESIGN CONSTRAINTS

⚠️ DO NOT MODIFY:
- Business logic
- Feature structure
- Routing
- Data flow
- Component functionality

✅ ONLY MODIFY:
- Layout
- Spacing
- Typography
- Color usage
- Visual hierarchy
- UI clarity

---

## Anti-Patterns to Avoid (VERY IMPORTANT)

- Gradient text for headings or numbers
- Glassmorphism / heavy blur effects
- Decorative blobs or glowing elements
- Repeated KPI sections showing same data
- Overloaded hero sections
- Auto-moving UI without user control
- “Generic SaaS AI dashboard” style

---

## Preferred UI Patterns

- Single strong hero message
- One clear proof section (NOT repeated metrics)
- Simple cards with clear purpose
- Large readable text
- Clear CTA hierarchy

---

## Color Usage

- Green → primary actions (agriculture, success)
- Teal/Blue → intelligence, weather, system
- Red/Orange → warnings only

Avoid using color for decoration only — it must convey meaning.

---

## Typography Rules

- Use ONE font family only (Inter / system UI)
- No gradient text
- Clear hierarchy:
  - Title: strong and short
  - Body: simple and readable
- Avoid long paragraphs

---

## Motion & Interaction

- Motion must guide attention, not decorate
- Avoid auto-rotating or auto-changing content
- Respect `prefers-reduced-motion`
- Keep animations subtle and fast (<300ms)

---

## Content Guidelines

- Avoid technical jargon (e.g. "PWA", "AI confidence")
- Use real-world language:
  - "Reduce crop loss"
  - "Spray before rain"
- Focus on outcomes, not technology

---

## Accessibility

- High contrast text
- Large touch targets
- Clear focus states
- No reliance on color alone

---

## Success Criteria

A good UI should:
- Feel like a real mobile app
- Be understandable in under 3 seconds
- Guide the user clearly to install or act
- Feel trustworthy and not over-designed
